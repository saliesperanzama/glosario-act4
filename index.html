Glosario de comandos GIT

Comando Breve Descripción Parámetros y Descripción Ejemplo
git init Inicializa un nuevo repositorio Git en el directorio actual. - Ninguno git init
git clone Clona un repositorio remoto en un directorio local. - URL del repositorio remoto. git clone https://github.com/ejemplo/repo.git
git add Agrega cambios al área de preparación (staging). - Nombre de archivos o directorios para agregar. git add archivo.txt
git commit Confirma los cambios en el repositorio local. - `-m "Mensaje"`: Mensaje de confirmación. git commit -m "Agrega cambios de funcionalidad"
git push Sube los cambios confirmados a un repositorio remoto. - Nombre del control remoto. - Nombre de la rama. git push origin master
git pull Descarga y fusiona los cambios desde un repositorio remoto al repositorio local. - Nombre del control remoto. - Nombre de la rama. git pull origin master
git fetch Descarga cambios desde un repositorio remoto sin fusionarlos. - Nombre del control remoto. git fetch origin
git branch Muestra una lista de ramas locales. - Ninguno git branch
git checkout Cambia a una rama específica o punto en la historia. - Nombre de la rama o identificador del commit. git checkout nueva-caracteristica
git merge Fusiona cambios de una rama a otra. - Nombre de la rama a fusionar. git merge caracteristica-a-fusionar
git status Muestra el estado actual de los archivos en el directorio de trabajo. - Ninguno git status
git log Muestra un registro de confirmaciones (commits). - Opciones para personalizar la salida (por ejemplo, `--oneline`). git log --oneline
git remote Muestra una lista de los controles remotos configurados. - Ninguno git remote
git rm Elimina archivos del directorio de trabajo y del repositorio. - Nombre de archivos para eliminar. git rm archivo.txt
git tag Administra etiquetas para marcar confirmaciones importantes. - `-a nombre -m "Mensaje"`: Crea una etiqueta anotada. git tag -a v1.0 -m "Versión 1.0"
git diff Muestra las diferencias entre el área de preparación y el directorio de trabajo. - Ninguno git diff
git reset Restablece el HEAD a un commit específico. - Opciones para especificar el tipo de restablecimiento (por ejemplo, --soft, --mixed, --hard). git reset --hard HEAD~2
git rebase Reorganiza la historia de confirmaciones aplicando cambios de una rama a otra. - Opciones para personalizar el rebase (por ejemplo, `-i` para rebase interactivo). git rebase rama-origen
git stash Guarda temporalmente cambios no confirmados para trabajar en otra tarea. - `-u`: Incluye archivos no rastreados en el stash. git stash
git bisect Ayuda a buscar un commit específico que introdujo un error. - Pasos manuales para determinar si un commit es bueno o malo. git bisect start, git bisect bad, git bisect good
git remote add Agrega un control remoto a un repositorio local. - Nombre del control remoto. - URL del repositorio remoto. git remote add upstream https://github.com/ejemplo/repo.git
